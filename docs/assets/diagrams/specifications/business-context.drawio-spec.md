# draw.io Design Specification — Business Context Diagram

- File name: business-context.drawio
- Source context: docs/02-Business-Requirements/business-requirements.md

## Purpose
Show the business relationship between the education market, schools, operational pain, the EduSync solution, and the value created for school stakeholders and the SaaS business.

## Components
- Market and demand
  - Indian school education market
  - Private schools and coaching institutes
- Problem layer
  - Operational fragmentation
  - Unified digital operations platform
- EduSync solution layer
  - Administration
  - Academic operations
  - Fees and payments
  - Parent and staff communication
  - Dashboards and AI insights
- Outcome layer
  - Reduced manual work
  - Improved academic tracking
  - Better collection visibility
  - Higher parent trust
  - Data-driven leadership

## Layout
- Use a left-to-right flow with a central solution block.
- Position market and schools on the left.
- Place problem and solution in the middle.
- Place capability modules and outcomes on the right.
- Use rounded containers for each domain cluster.

## Connections
- Market -> schools -> operational fragmentation -> unified digital operations platform.
- Unified digital operations platform -> each capability domain.
- Each capability domain -> one business outcome.
- Keep the visual path simple and avoid overconnecting every node to every other node.

## Color Recommendations
- Market: lavender (#7C3AED)
- Schools and demand: soft blue (#3B82F6)
- Pain: coral (#EF4444)
- Solution: navy (#1D4ED8)
- Capability domains: indigo (#4338CA)
- Outcomes: emerald (#10B981)
- Background: very light gray (#FAFAFA)

## Swimlanes
- Recommended swimlanes: Market, Problem, Solution, Value
- A four-lane structure will make the directional flow easier to follow.
